B08 核心时序模型实施计划

这份计划按 Superpowers plan 标准收束为可执行任务清单。它只围绕核心时序基础模型：模拟数据、模型 IO、基线、开源适配、模型路线决策和知识成果，不展开完整预测性维护系统。

主目标定义、原型化并评估 B08 核心设备时序基础模型。
数据驱动先模拟真空速凝炉完整时序场景，再倒推模型输入输出。
模型路线用同一任务集比较直接引用、轻量微调和领域自训。
知识成果专利、论文、数据集、benchmark 和模型资产作为一等产出。

2. 任务分解

Task 1：项目骨架与配置

pyproject.toml、src/b08_model_core/config.py、configs/furnace_fu13_sim.yaml

  • 定义设备、阶段、传感器、物理域、采样周期。
  • 测试：pytest tests/test_config.py -v

Task 2：观测数据 schema

tasks/schema.py

  • 冻结字段：timestamp、device_id、batch_id、stage、sensor_id、value、unit、domain、quality_flag、degradation_label、failure_proxy。
  • 测试：pytest tests/test_simulation_schema.py -v

Task 3：炉次与阶段时间线模拟

simulation/furnace_scenario.py

  • 生成 45 天、约 160 炉次，包含 8 个阶段和不规则停机。
  • 测试：pytest tests/test_furnace_scenario.py -v

Task 4：多物理域信号生成

simulation/signal_generators.py

  • 生成机械、液压、热、气氛、电气、流体六类信号。
  • 测试：pytest tests/test_signal_generators.py -v

Task 5：退化与故障代理注入

simulation/degradation_modes.py

  • 注入真空泄漏、冷却效率下降、液压密封衰退、泵振动上升、绝缘劣化。
  • 测试：pytest tests/test_degradation_modes.py -v

Task 6：模拟数据导出 CLI

simulation/export_dataset.py、cli.py

  • 命令：b08-model-core simulate --days 45 --seed 42 --output data/simulated/furnace_fu13_45d.parquet
  • 验收：至少 120 个炉次、15 个传感器。

Task 7：窗口与任务数据集构建

tasks/window_builder.py、tasks/labels.py

  • 生成 forecasting、imputation、reconstruction、representation、degradation 任务窗口。
  • 测试：pytest tests/test_window_builder.py -v

Task 8：模型 IO 规格与输出头

model_io/io_spec.py、model_io/output_heads.py

  • 定义 X、mask、delta_t、stage_token、sensor_token、domain_token、device_token。
  • 测试：pytest tests/test_model_io_spec.py -v

Task 9：基线模型与指标

baselines/、evaluation/metrics.py

  • 实现阶段感知稳健预测、重构 baseline 和评测指标。
  • 测试：pytest tests/test_baselines.py -v

Task 10：开源模型适配矩阵

adapters/base.py、evaluation/open_source_matrix.py

  • 记录 TSPulse、MOMENT、TTM、Chronos、TimesFM、UniTS、Moirai 的任务支持和路线建议。
  • 测试：pytest tests/test_open_source_matrix.py -v

Task 11：轻量 adapter 原型

adapters/ttm_adapter.py、moment_adapter.py、chronos_adapter.py、timesfm_adapter.py

  • 外部依赖必须可选；无权重下载时也能给出 unavailable reason。
  • 测试：pytest tests/test_open_source_matrix.py -v

Task 12：Benchmark runner

evaluation/benchmark.py、reports/model_core_evaluation.md

  • 输出模型、任务、指标、baseline 对比、路线推荐和原因。
  • 命令：b08-model-core benchmark --dataset data/simulated/furnace_fu13_45d.parquet --output reports/model_core_evaluation.md

Task 13：模型路线决策报告

docs/model-route-decision.html、reports/model_route_decision.md

  • 冻结直接引用、轻量微调、领域自训的进入条件和退出条件。

Task 14：知识成果规划

knowledge/patent_candidates.py、knowledge/paper_candidates.py、docs/knowledge-output-plan.html

  • 至少 3 个专利候选和 3 个论文候选，每项包含 novelty 和 evidence。
  • 测试：pytest tests/test_knowledge_outputs.py -v

Task 15：HTML 评审材料

docs/*.html

  • 确保 HTML 是主评审入口，Markdown plan 是执行入口。
  • 验证所有本地链接可解析。

Task 16：最终验证门

全项目

  • 运行 pytest -v。
  • 运行模拟数据生成和 benchmark。
  • 搜索验证模型核心关键词覆盖。

3. 路线决策标准

路线进入条件退出条件最小证据
直接引用zero-shot 或 frozen 表征优于稳健 baseline不能表达阶段/物理域上下文或部署过重预测、补全、embedding 或异常响应指标
轻量微调开源 backbone 有帮助，但领域偏差明显微调收益小或不稳定linear probe、adapter 或 LoRA 带来稳定提升
领域自训开源模型不满足核心 IO 或退化任务数据量或算力不足自定义预训练目标优于 baseline 且具备研究新颖性
