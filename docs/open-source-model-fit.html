开源时序模型适配判断

目标不是列模型清单，而是判断哪些开源模型可以直接引用、哪些适合微调、哪些只能作为 baseline。

1. 候选模型

模型链接优先评测能力B08 适配判断
TSPulseHugging Face表征、异常、预测、补全优先评估是否可作为多任务 backbone
MOMENTGitHubforecasting、imputation、classification、anomaly优先评估 embedding 和重构/异常任务
TTMHugging Face轻量预测、微调、多变量优先评估内网部署和预测残差任务
ChronosGitHubzero-shot / few-shot forecasting适合作为预测路线 baseline
TimesFMGitHub长上下文 forecasting、分位数适合趋势预测和分位数评测
UniTSGitHub统一多任务时序模型适合作为研究对照
Moirai / Uni2TSGitHub概率预测适合作为 forecasting 对照

2. 评测模式

模式做什么通过条件
Zero-shot不训练，直接在模拟数据和真实片段上预测显著优于稳健统计 baseline
Frozen embedding冻结模型，抽 embedding 做阶段/退化聚类阶段、退化和设备状态可分
Linear probe少量标签训练轻量头弱标签风险事件可识别
Adapter/LoRA轻量微调阶段和传感器适配比 frozen 模式稳定提升，且部署成本可接受
如果没有模型能处理阶段 token、多物理域传感器和退化前兆任务，B08 才进入自训小型领域 foundation model 路线。
