B08 模型核心 Brainstorming 成果

本页按你的反馈重新收束：B08 的主体不是预测性维护系统，而是其中的核心时序基础模型。外部项目需求只用于倒推模型输入、输出、功能边界和开源可引用性。

1. 重新定位

课题对象：设备时序基础模型本体。

直接目的：用一个具体设备场景的数据模拟和任务定义，判断模型应接收什么、输出什么、具备什么能力，以及现有开源模型能否直接引用、轻量微调或需要自训。

最终成果：不只是项目模块，还包括模型方法、数据集/评测基准、专利点、论文方向和可复用知识资产。

2. 两个核心工作

工作 A

模拟完整设备时序数据

以真空速凝炉为首个具体场景，模拟炉次、阶段、传感器、多物理域耦合、退化和潜在故障前兆。数据不是为了展示，而是为了倒推模型 IO、任务和评测方式。

工作 B

确定模型路线

围绕同一批模拟数据和真实数据切片，比较直接引用开源模型、领域微调、从零训练三条路线，判断哪条最符合 B08 的课题定位和交付节奏。

3. 模型 IO 初步定义

层级输入输出用途
基础预测阶段窗口内多传感器序列、mask、时间间隔、阶段 token、传感器 token未来序列、分位数、置信区间判断正常轨迹与未来偏离
表征学习单传感器/子系统/设备级窗口embedding、相似窗口、状态簇支持检索、冷启动和迁移
异常与退化预测残差、重构误差、历史基线、物理阈值异常分数、退化分数、变点、趋势斜率形成损坏前兆证据
任务适配少量故障、维修、专家复核标签风险概率、RUL、阶段级健康评分后期从基础模型走向维护预测

4. 开源模型判断标准

问题判断方式可能结论
能否直接用在模拟数据和真实片段上做 zero-shot / frozen embedding 评测可直接用于 forecasting、embedding 或 anomaly score 的某一部分
是否需要微调比较 LoRA/linear probe/adapter 前后效果若阶段语义和传感器分布差异大，优先轻量微调
是否需要自训当开源模型无法处理阶段、多物理域、退化前兆或内网部署约束时启动自训目标应是小型领域 foundation model，而非泛化大模型

5. 知识成果方向

  • 专利：阶段感知多物理域设备时序基础模型、基于模拟退化数据的预损坏表征学习、无故障标签下的设备损坏前兆评测方法。
  • 论文：工业小样本设备时序基础模型、仿真驱动的预测性维护预训练任务、工艺阶段条件化的多变量时间序列表征学习。
  • 数据资产：真空速凝炉模拟数据生成器、阶段化窗口数据集、退化注入基准、开源模型适配评测榜单。
结论：brainstorming 阶段已收束为“数据模拟驱动模型定义 + 开源/微调/自训路线评估 + 知识成果沉淀”。
