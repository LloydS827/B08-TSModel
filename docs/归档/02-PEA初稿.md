# B08 设备时序基础模型 PEA 初稿

版本：v0.1  
日期：2026-05-30

## 1. 问题定义

设备预测性维护项目需要提前判断设备或关键子系统是否存在损坏风险，并给出可执行的维护依据。当前已有真空速凝炉的多传感器时序数据、PLC 工艺阶段信号和设备配置，但缺少可复用的底层时序预测与风险推演模型。

真实困难包括：

- 故障标签缺失，不能直接训练监督故障分类模型。
- 单台设备历史数据少，不能依赖大样本训练。
- 采样存在停机间隔和非均匀性。
- 工艺阶段强相关，传感器含义随阶段变化。
- 多物理域耦合，损坏风险需要综合判断。
- 交付系统需要解释证据，而不是单一黑箱分数。

因此，本课题要解决的问题是：

> 如何在无故障标签、小样本、阶段强相关的工业设备场景中，建设一个可被预测性维护项目调用的时序模型底座，用于提前输出设备损坏风险、ETA 和解释证据。

## 2. 目标定义

目标用户：内部算法研发人员和预测性维护应用研发人员。

唯一目标：

> 形成一个可在预测性维护项目中调用的设备时序预测与预损坏风险推演模型底座，使研发团队能够基于真实设备时序数据提前输出设备或关键子系统的损坏风险、预计风险窗口和解释证据。

实施约束：

1. 核心计算支持内网部署，不依赖云端 API。
2. 输出必须可解释，能说明风险来自哪些传感器、阶段、趋势或阈值证据。
3. MVP 优先服务真实项目交付，不追求一开始跨全部设备泛化。

## 3. 方案描述

上线后的研发使用体验：

1. 研发人员配置设备、传感器、物理域、工艺阶段和阈值。
2. 系统从 InfluxDB 或离线数据集中读取历史时序。
3. 数据处理模块按炉次、阶段和时间窗口生成模型样本。
4. 模型输出未来轨迹预测、置信区间、重构误差和状态表征。
5. 风险推演模块融合残差、趋势、变点、阈值距离和跨传感器一致性。
6. 系统生成结构化风险结果，供诊断报告、日报、前端图表和工单模块使用。

最小闭环：

```text
真实设备时序数据
→ 阶段切片与样本构建
→ 时序模型预测/重构/表征
→ 残差与趋势证据
→ 预损坏风险评分与 ETA
→ 研发可调用 JSON 输出
```

方案差异：

- 不把异常检测当作最终目标，而是把异常、趋势和残差作为损坏风险预测的证据。
- 不在无标签阶段强行训练故障分类器，而是先输出可校准的预损坏风险。
- 不只做单传感器阈值报警，而是按工艺阶段和物理子系统融合多证据。
- 不把 LLM 作为核心判断模型，只把 LLM 放在结构化结果转写和诊断摘要生成环节。

## 4. 风险与措施

| 风险 | 影响 | 措施 |
| --- | --- | --- |
| 无真实故障标签 | 早期无法证明真实故障概率 | 使用专家复核、物理阈值事件、维修记录、合成退化和在线影子运行建立验证集 |
| 数据量不足 | 模型过拟合或不稳定 | 采用开源预训练模型、轻量微调、设备级基线和阶段化建模 |
| 误报过多 | 影响项目交付信任 | 使用持续性规则、置信区间、共形校准和多传感器一致性过滤 |
| 工艺阶段数据质量差 | 模型上下文错误 | 对 PLC 阶段信号做清洗和状态机校验 |
| 解释不足 | 运维和项目团队无法接受 | 输出残差、趋势、阈值距离、阶段和传感器证据 |
| 模型与系统脱节 | 算法成果不能交付 | 从第一版开始定义 JSON Schema 和接口边界 |

继续条件：

- 能稳定生成阶段化训练样本。
- 至少一个子系统能输出可解释风险结果。
- 项目团队认为输出对维护判断有帮助。

收窄条件：

- 如果多变量模型效果不稳定，先退回单传感器/单子系统模型。
- 如果真实故障验证不足，先以影子运行和专家复核作为 MVP 验证。

冻结或终止条件：

- 无法获得连续设备数据和阶段信号。
- 无法获得任何专家复核或维护记录，导致风险输出长期无法校准。
- 输出无法稳定区分正常波动与需要关注的风险事件。

## 5. 资源评估

关键资源：

- 数据：至少 1-2 个月连续时序数据、PLC 阶段信号、设备配置和阈值。
- 业务：设备工程师或运维人员参与复核风险事件。
- 算力：本地 GPU 服务器用于模型评测、微调和推理。
- 工程：InfluxDB 数据读取、离线数据集生成、JSON 接口和可视化图表。
- 知识：故障知识库、维修记录、停机记录和物理安全边界。

## 6. 成果衡量

### 6.1 MVP 指标

- 数据闭环：能够从真实数据生成阶段化样本，覆盖主要传感器和阶段。
- 模型闭环：能够输出预测值、置信区间、残差、趋势和风险分数。
- 解释闭环：每条 WARNING 以上风险必须包含证据链。
- 工程闭环：结果能够以 JSON 形式被上层系统读取。
- 复核闭环：项目团队能对风险事件进行有效/无效标记。

### 6.2 发布指标

- 稳定运行：分钟级或批处理准实时分析可持续运行。
- 误报控制：同一设备同一子系统的无效高等级告警数量可控。
- 提前量：对复核确认的风险事件，系统能在维护动作或异常恶化前给出提示。
- 可维护性：模型版本、数据版本、配置版本可追溯。

### 6.3 长期指标

- 故障或维修事件积累后，能评估指定时间窗口内故障预测准确性。
- 能从单台真空速凝炉扩展到同类设备。
- 能沉淀设备健康数据集、评测基准和可复用模型组件。

## 7. 竞品与参考

参考方向包括开源时序基础模型、工业异常检测库和预测性维护案例。可借鉴其预测、重构、表征和不确定性估计能力，但 B08 的差异化在于：

- 面向工业设备预测性维护交付。
- 工艺阶段感知。
- 多物理域风险融合。
- 从无标签预损坏风险逐步演进到故障概率/RUL。

详细调研见 [开源时序基础模型调研](调研资料/开源时序基础模型调研.md)。
