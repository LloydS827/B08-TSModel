E1

CT2_layered_validation

E2

CT1_problem_definition

关联 P1_stage_sensor_encoding、P2_small_sample_pretraining，用 representation probe 支撑阶段/传感器编码和小样本适配。

E3

CT2_layered_validation / CT3_unified_schema_metric

关联 P5_multitask_health_evaluation，用重建任务补足多任务健康评估的技术效果样例。

E4

CT2_layered_validation / CT3_unified_schema_metric

关联 P4_real_open_data_fusion，用开放数据映射检查真实数据、开放数据和任务指标是否能放入统一证据框架。

E5

CT4_decision_gate / P1-P5

E5 必须汇总 P1_stage_sensor_encoding、P2_small_sample_pretraining、P3_weak_label_anomaly_signal、P4_real_open_data_fusion、P5_multitask_health_evaluation，并进入 C -> B 决策门。

Data and Label Audit

数据与标签核对

每个 evidence 执行前必须完成同一组核对项，缺项时只能记录为 planned、needs-review 或 No_Go_hold，不能补写结果。

source

数据来源、内部来源记录、官方来源或 primary source 必须可追踪。

license

内部使用边界、开放数据许可和训练使用边界必须记录。

schema

timestamp、device、sensor、stage、quality、unit、missing rule 或开放数据 schema 映射必须明确。

label

弱标签、probe label、mask policy 或专家复核状态必须声明来源和置信边界。

split policy

time、run、batch、unit、fault trajectory 或 process split 必须在结果前固定。

Decision Gate

CT4_decision_gate / C -> B 决策门

C -> B 决策必须基于 E1-E5 的产物或失败原因，且 primary_task、strong_baseline、minimum_gain、seed_policy、confidence_interval_policy、failure_conditions 必须在结果解释前固定。

Go_to_B_minimal_prototype

最小原型可以进入 B 阶段，但只限于已通过契约的任务、数据和指标。

Stay_in_C_adaptation

继续 C 阶段适配，补齐 split、baseline、label 或模型适配证据。

Knowledge_only_consolidation

只沉淀论文、专利、调研和风险材料，不启动 B 阶段工程原型。

No_Go_hold

当数据、许可、标签、基线或稳定性不足时暂停推进，并记录失败原因。

Boundary

禁止解释

本登记页只定义最小证据的阅读和执行入口。任何真实实验都必须先通过契约校验；本页不代表实验已完成，不代表 C -> B 已通过，也不代表论文贡献、技术效果或专利判断已经成立。
