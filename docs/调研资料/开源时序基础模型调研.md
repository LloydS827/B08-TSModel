# 开源时序基础模型调研

版本：v0.1  
日期：2026-05-30

## 1. 调研结论

公开时序基础模型大多优先解决 forecasting，少数同时覆盖 imputation、classification、anomaly detection 或 embedding。对 B08 来说，不能简单把某个开源模型当成完整预测性维护系统，而应把它们作为以下能力模块：

- 未来轨迹预测。
- 预测分布和不确定性估计。
- 正常行为重构。
- 状态表征 embedding。
- 异常残差和趋势漂移证据。

最终的设备损坏风险、ETA 和维护建议仍需要结合工艺阶段、物理阈值、多传感器一致性和业务规则完成。

## 3. 适配分析

### 3.1 与 B08 高度匹配的能力

- 短期预测：用于判断设备正常情况下未来轨迹。
- 分位数预测：用于计算置信区间和风险边界。
- 重构误差：用于发现当前窗口是否偏离正常。
- embedding：用于建立设备状态库和新颖度检测。
- 多变量建模：用于液压、热、气氛、流体等子系统。

### 3.2 不能直接满足的能力

公开模型通常不会直接提供：

- 工艺阶段感知。
- 设备损坏风险定义。
- 子系统级维护建议。
- 工业物理阈值约束。
- 与管理后台、工单、日报的接口。
- 无标签阶段的项目验收闭环。

这些必须由 B08 自己建设。

## 4. 推荐评测组合

### 4.1 第一组：预测残差

候选：

- TTM。
- Chronos。
- TimesFM。

评测：

- 单传感器未来窗口预测误差。
- 子系统多变量预测误差。
- 置信区间覆盖率。
- 残差对已知异常或专家关注片段的敏感度。

### 4.2 第二组：重构与表征

候选：

- MOMENT。
- TSPulse。
- 自编码器基线。

评测：

- 正常窗口重构误差稳定性。
- 异常注入响应。
- embedding 对阶段、设备、子系统状态的区分能力。

### 4.3 第三组：工程基线

候选：

- Merlion。
- Darts。
- 稳健统计、EWMA、CUSUM、变点检测。

评测：

- 可解释性。
- 部署成本。
- 推理速度。
- 与规则层融合难度。

## 5. 选型建议

MVP 阶段建议采用“三层候选”：

1. 稳健统计和工程基线作为最低可交付能力。
2. TTM 或 Chronos/TimesFM 作为预测残差模型。
3. MOMENT 或 TSPulse 作为表征与异常检测模型。

不建议 MVP 一开始训练完全自研基础模型。更合理的路线是：

- 先用开源模型验证数据和任务可行性。
- 再围绕设备阶段、物理域和风险融合做领域适配。
- 积累足够多设备和事件后，再考虑自研预训练或专用 foundation model。

## 6. 与设备损坏预测的关系

开源模型解决的是“时序模式学习”，不是“设备损坏定义”。B08 需要在其上增加风险推演层：

```text
开源时序模型输出
→ 预测残差 / 置信区间 / 重构误差 / embedding
→ 阶段化正常基线
→ 趋势漂移、变点和阈值距离
→ 多证据融合
→ 预损坏风险、ETA 和解释证据
```

这条路线既能利用公开模型能力，又能确保最终结果服务于预测性维护项目交付。
