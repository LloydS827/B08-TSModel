# B08 Core Timeseries Model Brainstorming Design

## 定位

B08 的主体是核心设备时序基础模型，不是完整预测性维护系统。外部预测性维护项目只用于倒推模型输入输出、功能边界和开源模型适配判断。

## 用户

直接用户是内部算法研发人员。应用研发和预测性维护项目团队是模型结果的下游使用方。

## 核心工作

### 工作 A：模拟完整场景数据

以真空速凝炉 FU13 为首个场景，模拟 45 天、约 160 个炉次的多传感器数据。数据包含 PLC 阶段、设备停机间隔、多物理域传感器、数据质量标记、早期退化、风险事件和故障代理标签。

模拟数据用于倒推：

- 模型输入张量。
- 阶段、传感器、物理域、设备 token。
- 预测、补全、重构、表征、异常、退化、RUL 等输出头。
- 开源模型评测任务。

### 工作 B：确定模型路线

围绕同一套模拟数据和真实数据切片，评估三条路线：

1. 直接引用开源模型。
2. 开源模型轻量微调。
3. 自训小型设备时序基础模型。

## 模型输入输出

输入：

- `X[B,L,C]`：多传感器窗口。
- `mask[B,L,C]`：缺失、停机、异常采样标记。
- `delta_t[B,L]`：非均匀采样时间间隔。
- `stage_token[B,L]`：PLC 工艺阶段。
- `sensor_token[C]`：传感器身份与量纲。
- `domain_token[C]`：机械、液压、热、气氛、电气、流体。
- `device_token[B]`：设备身份。

输出：

- forecasting：未来序列、分位数、置信区间。
- imputation：缺失点补全。
- reconstruction：窗口重构。
- representation：窗口、传感器、子系统、设备 embedding。
- anomaly/degradation：异常分数、退化分数、变点和趋势。
- adaptation head：少量标签下的风险概率或 RUL。

## 候选模型路线

- TSPulse、MOMENT：优先评估表征、补全、异常和多任务能力。
- TTM、Chronos、TimesFM、Moirai：优先评估 forecasting 和不确定性。
- UniTS：作为统一任务模型参考。

## 知识成果

- 专利：阶段感知多物理域设备时序基础模型；仿真退化数据驱动的预损坏表征学习；无故障标签下的设备损坏前兆评测方法。
- 论文：工业小样本设备时序基础模型；仿真驱动的预测性维护预训练任务；工艺阶段条件化多变量时间序列表征学习。
- 数据资产：真空速凝炉模拟数据集、阶段窗口数据集、退化注入 benchmark、开源模型适配榜单。

## 收束结论

下一步计划只围绕模型核心展开：模拟数据、模型 IO、模型候选适配、训练/微调/引用决策和知识成果，不展开完整系统开发。
